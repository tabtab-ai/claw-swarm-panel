# 开源 | claw-swarm：让 OpenClaw 安全上岗，为每位员工配备专属 AI 数字员工

---

## 一个真实的困境

某中型互联网公司的技术负责人小王，最近遇到了一个头疼的问题。

公司决定给全员配备 AI 编程助手（OpenClaw），原本是为了提升效率。但推进过程中，问题接踵而至：

- OpenClaw 需要安装在员工的办公电脑上，**公司代码和 AI 对话记录全部留存在本地**，一旦设备遗失或员工离职，数据安全完全失控；
- 每位员工的 OpenClaw 都要单独配置模型 API 密钥，**50 个人就要维护 50 份配置**，哪天要换模型供应商，IT 团队要逐台操作；
- 办公电脑性能参差不齐，**跑 AI 推理时风扇呼呼转，影响日常使用**；
- 每来一个新员工，IT 同学要到工位上手动安装、配置，**入职当天根本用不上**；
- 员工离职后，本地残留的代码上下文和 API 密钥**无法远程清理回收**。

这不是小王一个人的困境。对于绝大多数中小企业来说，把 AI 编程工具铺到全员的真正难题，不是"能不能用"，而是**安全可控、统一管理、降低算力成本**。

---

## 解法：claw-swarm

**claw-swarm** 是一套专为中小企业设计的开源项目，目标只有一个：**为企业里的每一位员工，配备一个专属的 AI 数字员工。**

每个数字员工拥有独立的工作空间、独立的数据存储、独立的 AI 访问凭据，分配过程在 500 毫秒内完成，闲置时自动休眠，员工离职时一键回收——IT 无需介入。

项目由两个部分组成：

| 组件 | 职责 |
|------|------|
| **claw-swarm-panel** | Web 管理控制台 + HTTP API，负责用户管理和数字员工实例的全生命周期操作 |
| **claw-swarm-operator** | Kubernetes 控制器，负责实例池维护、存储管理和生命周期调度 |

---

## 它是怎么做到的？

### 1. 每人一个专属数字员工，物理隔离

每位员工都有一个独占的容器实例，配上独立的持久存储卷（PVC）。A 员工看不到 B 员工的任何数据，即使是 IT 管理员，也只能通过审计日志查看操作记录，而不是直接访问员工的工作内容。

```
员工 A 专属数字员工 + 独立存储
员工 B 专属数字员工 + 独立存储
员工 C 专属数字员工 + 独立存储
```

物理隔离，不是逻辑隔离。

### 2. AI 访问凭据按人管控，离职即回收

claw-swarm 对接了 **LiteLLM**，在为员工分配数字员工实例的同时，自动为其生成一个专属 AI API 密钥。

- 员工入职时：自动创建 → 密钥绑定到该员工账号
- 员工离职时：调用一次 `/claw/free` → 密钥立即吊销

不再有"一把钥匙开所有门"的安全隐患。

### 3. 500 毫秒内完成分配

系统维护一个**预热实例池**，常态保持若干空闲容器就绪。

当 HR 系统或 Slack 机器人调用分配接口时，直接从池中取出一个已就绪的实例，完成标记和密钥注入，响应时间 < 500ms。

没有冷启动等待，新员工入职当天就能用上自己的数字员工。

### 4. 自动暂停与恢复，杜绝资源浪费

下班了？数字员工自动缩容至零，存储完好保留。
早上来了？一键恢复，上次的工作环境完整复现。

支持延迟暂停（例如：10 分钟后再暂停），避免短暂离开就触发休眠。

## 对接外部系统有多简单？

claw-swarm 提供标准的 REST API，支持 API Key 认证，可以无缝对接：

| 系统 | 触发时机 | 操作 |
|------|---------|------|
| HR / 入职系统 | 员工入职 | `POST /claw/alloc` — 自动分配数字员工 |
| 离职流程 | 员工离职 | `POST /claw/free` — 回收实例与凭据 |
| 钉钉 / 飞书机器人 | 用户首次对话 | `POST /claw/alloc`（幂等） |
| 监控系统 | 定期巡检 | `GET /claw/instances` |

入职一行代码，专属数字员工自动就绪。

## 快速上手

项目完全开源，支持 Helm 一键部署：

```bash
# 1. 先部署 Kubernetes 控制器
# (参考 claw-swarm-operator 仓库)

# 2. 部署管理面板
helm install claw-swarm-panel oci://registry-1.docker.io/tabtabai/claw-swarm-panel-chart \
  --namespace tabclaw --create-namespace \
  --set ingress.hosts[0].host=claw-panel.example.com \
  --set ingress.className=kong

# 3. 本地开发模式
make run          # API 服务，端口 8088
make run-webui    # Web UI 开发服务器，端口 5173
```

默认管理员账号：`admin` / `happyclaw`（首次登录须修改）
